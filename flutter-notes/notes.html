3. Multi-Screen Apps Using Routes and Navigation

In flutter, "screens" and "pages" are called "routes".


Navigator.push() 
  Navigates to specified page. Requires a context and a MaterialPageRoute(builder: (context) => SecondRoute())
Navigator.pop() 
  Pops the current page from navigation and goes back to the previous screen.
MaterialApp property "routes" 
  Used for named routes, specified with a Map<string, (BuildContext) => Widget>
 Note that if you have specified the "home" property of MaterialApp, you can't specify a route with path '/' because it would be redundant.
Also note that if you have named routes starting with '/', there has to be one (and only one) route with the name '/'.
Set "initialRoute" for a different starting screen.

When navigating using named routes, consider putting the route String inside its related class as a static const String property, e.g.:
class WelcomeScreen extends StatefulWidget {
    static const String id = 'welcome_screen';
}

And then accessing it through WelcomeScreen.id, so initialRoute: WelcomeScreen.id,


To send context forwards during navigation (e.g. Navigator.push etc), pass it in the constructor of the new page, i.e. SecondRoute(data)
Passing arguments to a named route 
  https://flutter.dev/docs/cookbook/navigation/navigate-with-arguments

To send context backwards during navigation (e.g. Navigator.pop etc), pass it as the second value, i.e. Navigator.pop(context, cityName)
To retrieve the value from a pop(), await a value during the initial Navigator.push() -
it returns a Future, which is awaitable, i.e.:
var result = await Navigator.push(*route*);


Read more about navigation 
  https://flutter.dev/docs/cookbook/navigation/navigation-basics

10. Animations

Hero animations 
  a short animation of an icon (or similar) that exists on two sequential pages and moves between them

To create hero animations, you need 2x Hero widgets, 1x shared tag property, Navigator-based screen transitions
  1. Wrap the initial icon (the icon is likely already in a Container) in a Hero Widget
  2. Set the tag on the Hero Widget so that the two Hero Widgets can identify each other
  3. Setup navigation to the next screen
  4. Wrap the icon on the next screen in a Hero Widget
  5. Set the tag of the second Hero Widget to the same as the first
  6. Add the animation and icon to more screens if wanted

 Read more about hero animations: https://flutter.dev/docs/development/ui/animations/hero-animations

Custom animations with the Animation Controller
To create a custom animation, you need a Ticker, an Animation Controller and an Animation Value
  1. Add the mixin SingleTickerProviderStateMixin (or TickerProvider~ for multiple animations) to your State class (by using the keyword 'with')
  2. Add an AnimationController variable to your State class of your StatefulWidget, without initializing it
  3. Override initState() and inside the method, initialize the AnimationController (see next step for how)
  4. Required arguments are duration (takes a Duration object) and vsync (takes a widget that it syncs with, so use the 'this' keyword to use the instance of the State class)
  5. vsync can also be seen as 'the class that provides the ticker object'
  6. Call controller.forward() (where controller is an AnimationController instance)

12. Themes

Themes can be specified with the theme property, using ThemeData(*theme-properties*), the docs are a good resource for more info

If you'd like to use the default dark theme and only change a few specific parts of it, you can use:
ThemeData.dark().copyWith(*theme-properties*)

 To set a custom theme for one individual widget, you can wrap the widget in a Theme widget,
 set the data property of the Theme widget to a custom ThemeData,
 and the child of the Theme widget to the individual widget you want the theme to be used on
 e.g. Theme(data: *theme*, child: FlatButton()) will use a custom theme that only effects that FlatButton

Copying a theme 
  For some theme types it's possible to copy the closest theme data by accessing [Widget]Theme.of(context).copyWith(*changes*)
 e.g. SliderTheme.of(context).copyWith(thumbShape: RoundSliderThumbShape(enabledThumbRadius: 15.0))

14. Structuring projects

Separate out pages to their own dart files
Sample file structure (lib/)
  * components/reusable_card.dart (custom widget)
  * components/other_widget.dart (custom widget)
  * screens/input_page.dart (page widget class)
  * screens/other_page.dart (page class)
  * services/weather.dart (service class)
  * services/networking.dart (service class)
  * utilities/constants.dart (project constants)
  * main.dart

Import a file in the lib folder from a subfolder 
  import 'package:app_name/constants.dart';

main.dart is mainly used for setting the theme of the app,
 and constructs a widget from another file, e.g. input_page.dart,
 which is set in the home property of the MaterialApp widget

main.dart structure 
  *imports*
  *main() calling runApp()*
  *app widget class containing theme data, the widget that is called by runApp()*

18. Sizing

- When widgets should take up as much space as possible inside a Column or a Row, wrap the widget with an Expanded widget
- When working with Rows and Columns, unless mainAxisSize is specified to MainAxisSize.min, they will try to take up as much 
horizontal/vertical space as they can, even if not required
- Setting width or height to double.infinity will make them take up 100% of the space available to them
- CrossAxisAlignment.stretch will stretch children of a Column/Row to take up 100% of the space available to them
without requiring you to set the width/height properties on the children
- IntrinsicWidth/IntrinsicHeight 
  if you want to set the width of all items in a Column to the largest item's width,
  wrap the Column inside an IntrinsicWidth
- To get the max width/height, wrap your widget in a LayoutBuilder

25. Function delegates

The "Function" type is a function delegate (i.e. basically Action in C#), can be used like this:
int calculate(int n1, int n2, Function calculation) {
   return calculation(n1, n2);
 }
 // call calculate()
 print(calculate(10, 15, add));
 print(calculate(10, 15, multiply));

 int add(int n1, int n2) => n1 + n2;
 int multiply(int n1, int n2) => n1 * n2;

And it can also be assigned to a variable:
// assign Function to a variable (end with semicolon even if using curly braces):
 Function calculator = (int n1, int n2, Function calculation) => calculation(n1, n2);
 print(calculator(1,2,add));


26. constness/finalness

The "final" keyword acts as readonly in C#, it is possible to set it through the constructor but not after the constructor has run,
 i.e. it can only be set once
e.g. final int x = 1;
or final int x;

The "const" keyword acts almost as const in C#, it is a compile-time constant;
  e.g. const int x = 1; // compile-time constant, can't be changed
 however, it is also used to canonicalize values (see *2), so that these objects:
  Point p1 = const Point(0, 0);
  Point p2 = const Point(0, 0);
 are the same object - only one object is created here, as seen by using identical(p1, p2); // returns true
 but these are not:
  Point p3 = const Point(0, 0);
  Point p4 = const Point(0, 1);

Putting the "const" keyword before a constructor ensures that the constructor does not have a body,
 and that all fields in the class must have the "final" keyword (i.e. they have to be final), so that they can be canonicalized

To have a class-wide const you have to make it static as well, e.g. static const String test = 'fest';

28. Stateful widgets

Lifecycles
Stateful widgets' State<T> base classes have three main methods; initState(), build(), deactivate()
These handle the lifetime of the widget and they can all be overridden by the stateful widget

initState() is called when the widget is initialized - if possibe,
put resource intensive code here (as initState is only called once)

build() is called after initialization and whenever the widget's state is updated (using setState() for example)

deactivate() is called when the widget is destroyed (e.g. if switching to another screen)

Inheritance
Consider a stateful widget for a screen, 'LocationScreen'
LocationScreen extends the StatefulWidget and initializes a new instance of its accompanied State class ('_LocationScreenState')

_LocationScreenState extends State<LocationScreen>, which gives it access to the lifecycle methods (initState, build, deactivate),
but just as importantly, it gets access to the 'widget' property, which points to the LocationScreen instance

When we need to initialize a widget with some state passed through the constructor of LocationScreen,
we can access that state in the _LocationScreenState by tapping into the 'widget' property and finding the property we're looking for inside the LocationScreen instance.

Example:
class LocationScreen extends StatefulWidget {
  LocationScreen({this.locationWeather});
  final locationWeather;
  @override
  _LocationScreenState createState() => _LocationScreenState();
}

To use a class from another file, import it using "import 'fileContainingClass.dart';"
To make a field private, prefix it with an underscore (_), e.g.

To have a constructor which takes an argument that is named the same as a field, you can assign the value to it by accessing it through
"this", e.g. "this.text = text;"

Different types of constructors:
class Question {
String text;
bool answer;
/* constructor type 1: */
Question({String q, bool a}) { 
text = q;
answer = a;
} // Question(q: 'Is true?', a: true);

/* constructor type 2, equivalent to 1: */
Question({this.text, this.answer}); // Question(q: 'Is true?', a: true);

/* constructor type 3: */
Question(String q, bool a) {
text = q;
answer = a;
} // Question('Is true?', true);

/* constructor type 4, equivalent to 3: */
Question(this.text, this.answer); // Question('Is true?', true);

/* constructor type 5 (named constructor): */
Question.meaningOfLife() {
text = "Is there a meaning to life?";
answer = true;
} // Question.meaningOfLife();
}

Named constructor:
class Point {
num x, y;
Point(this.x,this.y);

Point.zero() { // named constructor - called through Point.zero();
x = 0;

}
}

  A process for converting data that has more than one possible representation into a "standard" canonical representation.
